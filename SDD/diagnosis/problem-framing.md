# 问题诊断与边界说明

## 1. 真实问题

社区文字审核的困难不是缺少一个更长的敏感词表，而是风险依赖语境：

- 谁在说；
- 对谁说；
- 是本人攻击，还是引用他人内容；
- 是举报、反驳，还是赞同攻击；
- 单条是否安全，但连续行为是否构成骚扰；
- AI 是否有足够上下文和真实证据。

## 2. 主要利益相关方

| 角色 | 关注点 | 可能受到的伤害 |
| --- | --- | --- |
| 普通用户 | 正常发言不被误判 | 内容被错误限制、申诉无门 |
| 被针对用户 | 不被骚扰、威胁和群体施压 | 隐晦攻击被漏判 |
| 审核员 | 快速看懂案例并作出可信决定 | AI 理由错误或证据虚构 |
| 平台 | 社区秩序和审核效率 | 过度封禁或风险内容放行 |
| 开发团队 | 两天内完成可证明的 MVP | 范围失控、证据链缺失 |

## 3. 核心冲突

1. 自动化效率与误判风险；
2. 保护社区安全与保障正常表达；
3. 使用更多上下文与控制成本、隐私；
4. 展示 AI 能力与保留人工责任；
5. 做出完整社区与两天内完成审核闭环。

## 4. 约束

- 两天开发；
- 两名成员；
- 至少服务端 + Web 前端两个技术端；
- 只处理文本；
- MVP 允许预置身份；
- 必须有正常、边界、失败或误判、回归验证；
- 至少一条端到端验证；
- 至少三条题面之外的边界或失败样例；
- 必须保存 AI 采纳、拒绝、修改记录；
- 个人贡献必须能指向文件、代码或测试证据。

## 5. 非目标

- 不建设完整社交平台；
- 不实现多社区与成员管理；
- 不承诺生产级模型准确率；
- 不让 AI 自动完成申诉终审；
- 不追求复杂 Agent 数量；
- 不用页面数量代替问题判断。

## 6. 最小证明目标

MVP 只需可信地证明：

1. AI 使用了真实上下文，而非只看敏感词；
2. 系统校验了 AI 证据，而非盲信输出；
3. AI 不确定时能够安全转人工；
4. 申诉可以引入新信息并纠正一次误判；
5. 整个过程可以被回放和解释。

## 7. 当前不能证明什么

即使 MVP 全部通过，也不能证明：

- 对所有互联网文本都判断正确；
- 能替代真实平台的专业审核与法务流程；
- 能处理图片、语音、视频；
- 在大规模并发和复杂攻击下仍满足生产要求；
- 固定样例集上的效果等同于真实准确率。
