# 原型说明（Prototype Guide）

## 1. 原型目的

原型不用于证明完整社区能力，而用于证明：

1. 审核结果真实影响内容是否公开；
2. AI 能结合上下文判断；
3. AI 失败和不确定性会转人工；
4. 用户可以申诉；
5. 人工可以纠正误判；
6. 结果可以追溯。

## 2. 最小页面

普通用户：

- 社区与话题列表；
- 话题详情和线性楼层；
- 发起话题/发布楼层；
- 我的发布；
- 审核详情；
- 申诉页。

审核员：

- 待复核列表；
- 复核详情；
- 审核历史或时间线。

## 3. 原型边界

不包含：

- 多社区加入；
- 点赞、收藏、私信；
- 嵌套评论；
- 完整登录；
- 图片、视频和语音；
- 生产级模型准确率。

## 4. 模拟数据说明

允许使用：

- 预置用户；
- 固定社区；
- 固定话题；
- 固定边界样例；
- Mock Provider。

但必须在 README 明确：

- 哪些是 Mock；
- 哪些是真实 API；
- 哪些是真实数据库状态；
- Mock 只能证明流程，不能证明模型准确率。

## 5. 演示稳定性

准备：

- Seed 数据；
- 一键启动命令；
- 固定 Prompt/Provider 配置；
- 正常、骚扰、申诉改判三个固定案例；
- 截图；
- 录屏兜底；
- AI 不可用时的失败演示。

## 6. 原型完成标准

- 主路径可操作；
- 核心判断实际触发；
- 至少一个边界/失败；
- 输入、AI 建议、系统决定和人工结果可回溯。
